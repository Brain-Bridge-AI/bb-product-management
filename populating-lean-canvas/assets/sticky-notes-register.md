# Sticky-Notes Register

A scratch buffer for cross-box insights surfaced during the fill. Whenever working one box surfaces something that belongs to a different box, capture it here — don't interrupt the current box to jump ahead.

At close-out, sweep this register and place each item in its correct target box.

## Format

Each entry:

```
- <short insight>
  Source box: <where it came up>
  Target box: <where it belongs>
  Provenance: [heard] / [assumed] / [★]
  Notes: <any context or verbatim quote>
```

## Example entries

```
- The CFO is a fractional CFO who could resell Solo to his other engagements
  Source box: Customer Segments
  Target box: Channels (specifically: reseller channel)
  Provenance: [heard]
  Notes: Mentioned during Segments brainstorm — "he's a fractional CFO" + earlier context.

- Forward-leaning on technology is probably a better segment qualifier than company size
  Source box: Channels
  Target box: Customer Segments
  Provenance: [assumed]
  Notes: Surfaced while thinking about who in Vistage would be a good fit.

- "My chief of staff spends Fridays chasing people for updates"
  Source box: Problem (pains)
  Target box: Problem (pains) — verbatim quote
  Provenance: [heard]
  Notes: Captured during shoe-stepping prep; place in Problem box.
```

## When to use the register

- You're working Box A and an insight about Box B surfaces. Capture here, keep working Box A.
- The user produces a verbatim customer quote that fits multiple boxes. Capture here, decide target during close-out.
- A back-edit for an earlier box comes up while working a later box. Capture here — during close-out you apply the edit in-place.

## When NOT to use the register

- For items that belong in the current box. Put those directly in the box.
- For Go-learn-this items. Those go in the Go-learn-this list at canvas close-out, not here.
- For post-fill activities (validation plans, experiment design). Tell the user those are out of scope; don't even capture in the register.

## Close-out sweep

At canvas close-out, review the register:

1. For each entry, apply it to the target box.
2. Mark as "applied" once placed.
3. Any entry that now looks off-target or obsolete gets deleted (not all mid-fill insights survive scrutiny).
4. The register itself appears in the final canvas output as a short section, showing what cross-box bleed happened — useful for future canvases on similar products.
